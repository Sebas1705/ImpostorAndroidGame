# Security Policy
## Supported Branch
Security fixes are applied to the active `main` branch.
## Reporting a Vulnerability
Please do **not** open a public issue for security vulnerabilities.
Report privately by email to: `sebas1705@gmail.com`
Include:
- Affected area/module
- Reproduction steps
- Impact assessment
- Optional mitigation ideas
## Response Targets
- Initial acknowledgement: within 2 business days
- Triage and severity classification: within 5 business days
- Fix plan shared after triage
## Disclosure
After a fix is released, coordinated disclosure is welcome.
