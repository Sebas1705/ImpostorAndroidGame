# Docs Checklist

Use this checklist before merging documentation-heavy changes.

## Required

- [ ] `README.md` reflects current commands and key links.
- [ ] Policy docs align with current workflows and branch checks.
- [ ] AI context docs are updated when architecture or operations change.
- [ ] New files are referenced from at least one index (`README.md` or `AI_INDEX.md`).
- [ ] `docs/ai/ARCHITECTURE_GUARDRAILS.md` still matches current module dependency rules.
- [ ] `docs/ai/FLOW_CATALOG.md` contains at least one real, current end-to-end flow.

## Validation

- [ ] Commands shown in docs are syntactically valid for PowerShell.
- [ ] Paths point to existing files.
- [ ] Root-vs-subfolder references are explicit when needed.
